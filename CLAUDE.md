# CLAUDE.md

## Repository summary

Flyrank.ai is currently a capstone starter repository. The application stack has not been implemented yet, so this file documents the conventions to follow as the project grows.

## Current stack

- Git for version control
- Markdown for docs
- MIT License for the repository

## Conventions

- Use Conventional Commits for every commit message.
- Keep the README concise and update it when the project scope changes.
- Prefer small, focused changes over broad rewrites.
- Keep generated files, build outputs, and local environment files out of version control.
- Use descriptive filenames and avoid abbreviations in new project assets.

## Workflow

- Add implementation details only after the project scope is finalized.
- Document any new framework, runtime, or deployment choice in this file.
- When the stack is chosen, add setup, lint, test, and run instructions to the README.
